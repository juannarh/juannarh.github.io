El Modelo de Caja en CSS

Aprenderemos sobre el modelo de caja, las propiedades que maneja y cómo afectan su comportamiento.

Propiedades básicas de css

1: Display

Display toma muchos valores diferentes, pero solo 4 son los más utilizados.

div {
              bloqueo de pantalla;
              pantalla: bloque en línea;
              pantalla: en línea;
              pantalla: ninguno;
              }

- block: muchos elementos HTML se configuran en este modo de visualización mediante las hojas de estilo de los navegadores. Incluyen <div>, <ul> y bloques de texto como <p>. Los elementos de nivel de bloque por defecto ocupan tanto espacio como pueden, y no se pueden colocar en la misma línea horizontal con ningún otro modo de visualización, incluidos otros elementos de bloque. (Excepción: a menos que estén flotando) Con los elementos de bloque, obtienes la capacidad de alterar el ancho y la altura del elemento a tu gusto, por eso se usan para diseños

- inline: el modo en línea ajusta muchos elementos HTML a su alrededor y es el valor predeterminado para todos los elementos que no se especifican con ningún otro valor de visualización. Los elementos se pueden colocar uno al lado del otro en la misma línea que los elementos en línea. Piense en la etiqueta <strong> que en negrita elementos, la etiqueta <em> que crea cursiva y etiquetas <a> que le permiten vincular a otras páginas web. Todos estos son ejemplos de elementos en línea. No podrá cambiar el ancho y el alto de un elemento en línea.

- inline-block: este es un valor de visualización que combina las propiedades de los elementos de bloque y los elementos en línea. Tiene la capacidad de establecer una altura y un ancho, y el elemento puede aparecer en la misma línea horizontal que otros elementos.

- none: Display none oculta el elemento del sitio web y no se mostrará visualmente. Esto es muy útil para los menús desplegables de CSS, donde aparecen opciones adicionales al pasar el mouse sobre los menús de navegación. La razón es que los elementos se establecen en un valor de visualización de ninguno inicialmente, y el valor de visualización se cambia para bloquear al pasar el mouse.

Páginas de GitHub

Los planes de alojamiento web vienen en una variedad de opciones y puntos de precio. Hay planes más caros que tienen la capacidad de alojar los sitios web más grandes del mundo, pero ¿qué sucede si solo necesita una solución de alojamiento simple para un sitio web simple?

Para un sitio web estático o aplicaciones web pequeñas, existen planes de alojamiento gratuitos que pueden ponerlo en funcionamiento en línea en un momento. Requieren un poco más de configuración que un host pagado, pero es una compensación digna de forma gratuita.

GitHub Pages es una de esas opciones, y en este artículo le mostraremos cómo alojar un sitio web simple usando GitHub Pages de forma gratuita.

GitHub es una gran herramienta para almacenar proyectos y colaborar con otros, pero su utilidad no se detiene allí. Utilizaremos un servicio llamado GitHub Pages para compartir nuestras creaciones de páginas web en la World Wide Web.

GitHub Pages ofrece muchas características y flexibilidad, todo gratis. Algunos de los beneficios incluyen:

- Configuración fácil

- Colaboración perfecta con Git y GitHub

- Alojamiento gratuito con> 95% de tiempo de actividad

- Actualización en vivo con flujo de trabajo normal de GitHub

La implementación en las páginas de GitHub es automática. Una vez que está configurado, la implementación se realiza cada vez que transfiere los cambios locales a su repositorio remoto alojado en GitHub. Dirígete a las instrucciones de configuración de GitHub Pages y sigue los pasos exactamente para obtener la configuración de tu página principal de GitHub Pages.

Cuando navega por primera vez a su sitio recién implementado, es posible que reciba un error 404. Si esto sucede, y está seguro de haber seguido todos los pasos tal como están escritos, vuelva a revisar en 30 minutos para ver si la implementación se ha realizado correctamente.
